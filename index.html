HTML

  • Elements hold content. Attributes describe the characteristics of an HTML elemt in detail. Found in the starting tag.
  • The head element contains information about the webpage such as metadata.The head element is not visible to user.
  • The body element represents the visible contact shown to the user.
  • HTML attributes will always be in the beginning, never the ending.
  • The title of a webpage is important for SEO.
  • HTML elements can include text, images, headers, and lists.
  • HTML elements are case sensitive. Declarations are not case sensitive.
  • Meta element defines Metadata, information used by the browser.
  • Elements nested inside of other elements are known as children elements. Parent elements hold all children elements.
  • The h1 element is to only be used once per webpage, but you can have h1 to h6 on one webpage.
  • An element is inside of an opening and closing tag. A tag is what is inside angle brackets.
  • P tag is typically for paragraphs and text blocks. Where the main text content may be on your webpage.
  • ul tag is for a following unordered list.
  • ol tag is for an ordered list.
  • li tag is for list items.
  • src defines location of an image such as a URL.
  • br tag is to create a line of empty space or line break between two blocks of content.
  • a tag creates a link to either the same webpage or another webpage. href attribute points to the URL for the link.
  • alt attribute describes an image for screen readers. This can also improve SEO for your website.
  • Footer tag usually has author, copyright, contact, or navigation information.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Increasing the padding makes the entire element bigger.
  • Padding is transparent and it is inside an element.
  • DRY stands for do not repeat yourself

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git repo names cannot have spaces, you need to dash instead of space.
  • pwd will provide a present working directory. It is the folder or path that the bash session resides in.
  • ls will provide a list of files and folders in the current directory you are in.
  • cd changes directories to the folder/location you wish to be in.
  • cd.. will let you navigate backwords from the current directory to the parent directory one level at a time.
  • mk dir (new name) creates a new directory.
  • touch creates a new file inside a directory
  • start opens a file
  • clear will visually clear previous code, but keep you on the same path.
  • SSH keys allow you to interact with github without having to use a password every time.
  • github is a cloud based service used for storing, managing, and collaborating on codes.
  • A repo is where a project will reamin throughout its entire life cycle.
